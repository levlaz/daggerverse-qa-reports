Module Purpose

The kubeconform module is designed to validate Kubernetes resources against the Kubernetes OpenAPI specification. It provides functionality to:

  • Validate standalone YAML files and kustomization files
  • Exclude specific directories from validation
  • Convert CRDs into JSONSchemas
  • Support additional schemas from the Datree Catalog
  • Support Flux variables substitution

Available Functions

  • validate(): Validates Kubernetes manifests in the provided directory with optional CRDs
  • version(): Returns the Kubeconform version used for validation
